# PR Title
Phase 3 foundation: auth, shell, protected APIs, and Supabase migrations

# Overview
This PR introduces the Phase 3 foundation for Cornerstone by delivering a unified app shell, authenticated session and actor resolution, protected operational APIs, migration-backed data scaffolding, and documentation updates. Work is split into logical commits to keep review scope clear and traceable.

# What Changed

## Runtime and Platform
- Migrated request guard entrypoint from middleware to proxy.
- Stabilized localhost auth behavior and dev preview handling in the app frame.
- Resolved TypeScript and lint blockers in key routes and shell components.

## Auth and Access Control
- Added actor resolution and role normalization helpers.
- Added authenticated actor endpoint at /api/auth/me.
- Added sign-in, sign-out, and reset-password flows.
- Improved sign-in network error messaging for unreachable Supabase auth endpoints.

## UI Shell and Module Pages
- Added shared page shell, top navigation, protected banners, and branding components.
- Added and updated module pages for health and safety, safer recruitment, reports, registered manager, and operational dashboards.

## Protected APIs and Service Layer
- Added protected API handlers for aria, daily logs, handover, incidents, maintenance, oversight, reports, safer recruitment, and tasks.
- Added phase3 service layer and repositories for health-safety, recruitment, reports, incidents, tasks, and audit logging.

## Data and Migrations
- Added Supabase Phase 1 foundation migration and Phase 3 migration.
- Added SQL companion files for RLS and storage setup.
- Updated Prisma and TypeScript configuration for runtime support.
- Updated ignore rules for local artifacts (dev.db and supabase/.temp).
- Added npm override for @hono/node-server to apply a patched transitive dependency version.

## Documentation
- Updated README.
- Added client demo script.
- Added release notes for this rollout.

# Verification
- npm run lint passed.
- npm run build passed.
- Local routing and auth smoke checks passed on localhost:3001.
